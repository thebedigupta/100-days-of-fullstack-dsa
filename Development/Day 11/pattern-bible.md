# HTTP Headers and Fetch API

## HTTP Headers
Essential information that accompanies HTTP requests and responses.

### Request Headers
Custom labels sent with outgoing requests.
- May contain sensitive information (passwords, tokens)
- Security is managed by the browser
- Provide metadata and authentication details

**Example:**
```javascript
{ 'Authorization': 'Bearer token123', 'Content-Type': 'application/json' }
```

### Response Headers
Labels generated by the server.
- Identify the type of data received
- Contain server metadata (content type, caching info)
- Help process the server's response

**Example:**
```javascript
{ 'Content-Type': 'application/json', 'Set-Cookie': 'sessionId=abc123' }
```

## Making POST Requests with Fetch

### Fetch Options
- `method`: The HTTP method (e.g., 'POST')
- `headers`: Request header object
- `body`: Data to send (string, JSON, FormData, Blob, or URLSearchParams)

### Example: POST a User Object as JSON
```javascript
let user = { name: 'John', surname: 'Smith' };

let response = await fetch('/article/fetch/post/user', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json;charset=utf-8' },
  body: JSON.stringify(user)
});

let result = await response.json();
alert(result.message);
```

## Handling Responses

### Response Properties
- `response.status`: HTTP status code
- `response.ok`: True if status is 200-299
- `response.headers`: Map-like object of HTTP headers

### Reading Response Body
- `response.text()`: Plain text
- `response.json()`: JSON data
- `response.formData()`: FormData
- `response.blob()`: Blob object
- `response.arrayBuffer()`: ArrayBuffer

### Pattern: Check Response Before Processing
```javascript
const response = await fetch(url);
if (!response.ok) {
  throw new Error(`HTTP ${response.status}`);
}
const data = await response.json();
```
